Static Documents

Encrypt a File

Encrypt a PDF or image before committing it to your repository. The encrypted file is safe to host publicly — without your vault password it is unreadable ciphertext. After uploading, add "encrypted": true to its entry in documents.json.

03
Cross-Device Sync

Firebase Setup

Without Firebase, uploaded files only exist on the device that uploaded them. Complete this section once and every device will share the same vault. Takes about 10 minutes. Files are still encrypted before they reach Firebase — the server never sees your content.

1
Go to console.firebase.google.com, sign in with any Google account, and click Add project. Give it any name and disable Google Analytics.
2
In the left sidebar open Build → Firestore Database. Click Create database, choose Production mode, pick the region nearest you, and click Done.
3
Open Build → Authentication. Click Get started. Under Sign-in method enable Email/Password. Go to the Users tab and click Add user — enter any email and password. These are your Firebase credentials, separate from your vault password.
4
In Firestore Database → Rules, replace all content with the rule below, then click Publish. This covers both metadata and file storage.
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /{document=**} { allow read, write: if request.auth != null; } } }
5
Go to Project Settings (gear icon) → General tab → scroll to Your apps. Click the web icon </>, give it any name, click Register app. Copy the config values into the fields below.
Paste into js/config.js
Open js/config.js, find the firebaseConfig block, and replace it with the snippet above. Save the file and redeploy.
